# Chapter Quizzes & Checks for Understanding  →  built as a tool

> **Implemented.** Generated by `build-quizzes.py` at the repo root. Run after
> the spine is complete (GATE 6 signed).

**What it is:** a three-layer quiz suite per chapter — a **prequiz** (before
reading, short-answer, failure-primed), embedded **section checks** with
immediate feedback, and an **end-of-chapter quiz** (10–15 items) with the
Bloom distribution matched to the chapter type and **interleaved review** items
from prior chapters. Every stem/distractor/explanation is grounded in the
chapter text, not general knowledge.

**Inputs:** chapters in `chapters/` (+ a key in `.env`).
**Outputs:** one markdown file per chapter in `editions/quizzes/quiz-<stem>.md`.

## Run it

Calibration first (the spec's "generate one, review, then batch"), wired to the
verified-artifact gate:

```bash
python build-quizzes.py                 # calibration: first chapter only, then STOP
# review editions/quizzes/quiz-00-introduction.md against the 5-point checklist it prints
python scripts/verify.py sign editions/quizzes/quiz-00-introduction.md --by <name> --note "calibration ok"
python build-quizzes.py --all           # batch the rest (skips any already-verified quiz)
```

Other modes: `--chapter 04` (one chapter), `--limit N`, `--all --force` (skip the
calibration gate). Config in `quiz-config.yaml` (`skip_chapters`, `output_dir`,
`interleave_review`).

> Filenames use the chapter **stem** (`quiz-01-what-ai-plus-one-is.md`) rather
> than `quiz-chapter-NN.md`, because several chapters share a leading number
> (e.g. two `97-*` files) — the stem guarantees a 1:1 mapping with no collisions.
